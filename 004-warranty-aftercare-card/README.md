# 004 / 保証・アフターサービス案内カード

工事完了後の「保証年数」と「点検スケジュール」を1枚の証書にまとめた、建設業向けUIパーツです。

## コンセプト

001〜003が「実績を記録する」「変化を見せる」「工程を伝える」パーツだったのに対して、004は工事が終わったあとの安心感を伝えるパーツです。

- 右上の丸いバッジは、保証年数を刻んだ検査印（ハンコ）のモチーフです。二重の点線リングと傾いた文字で、押し当てたような質感にしています
- 点検スケジュールは点線でつながった時系列で表示し、完了した点検には「済」のハンコがスタンプされます。カードがスクロールで画面に入ると、済スタンプが左から順番に押されていく演出付きです
- 「予定」のノードはクリックすると「済」に切り替わり、その場でハンコが押されます。実際の運用では点検実施日をもとに `data-status` を自動で更新する想定のデモ操作です
- 上部の点線（ミシン目風）は証書らしい"控えを切り離す"雰囲気を出すための装飾です

配色・書体は001〜003と共通のトークン（クラフト紙×ブループリント、Shippori Mincho + Zen Kaku Gothic New）を踏襲し、シリーズとしての一貫性を持たせています。

## 使用技術

- HTML / CSS / JavaScript（フレームワークなし）
- `IntersectionObserver` でスケジュール表示のタイミングを検知し、CSSの `transition-delay` で済スタンプを順番に出現させています
- 点検ノードは `<button>` + `aria-pressed` で実装し、キーボード操作・スクリーンリーダーにも対応しています

## ファイル構成

```
004-warranty-aftercare-card/
├── index.html   … 保証書カードのマークアップ
├── style.css    … カード・保証印バッジ・点検スケジュールのスタイル一式
└── script.js    … スクロール検知と済/予定の切り替え処理
```

## 使い方

1. `.wc-card__title` や `.wc-card__spec-row`、`.wc-contact__row` の中身を、実際の工事内容・証書番号・連絡先に差し替えてください
2. `.wc-seal__num` / `.wc-seal__unit` を保証年数（例：「10」「年保証」）に合わせて変更してください
3. `.wc-check` の `data-status` に `done`（済）または `upcoming`（予定）を設定すると、初期表示の状態を切り替えられます。点検項目を増減する場合は `.wc-check` の `<li>` ごと追加・削除してください
4. 点検ノードはデモとしてクリックで済/予定を切り替えられますが、実運用では点検実施日と現在日時を比較して `data-status` をサーバー側またはJSで自動設定する形が想定用途です

## 対応

- キーボード操作（Tab + Enter/Space で点検ノードを操作可能）
- `prefers-reduced-motion` 設定時は済スタンプのアニメーションを簡略化
- 380px程度の狭い画面でもヘッダー・点検スケジュールが縦積みで崩れないレイアウト
